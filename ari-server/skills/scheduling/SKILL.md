# Scheduling Skill

일정 예약 및 알림 설정을 관리하기 위한 스킬입니다. 반복되는 작업이나 일회성 타이머/알림을 등록, 조회, 삭제할 수 있습니다.

사용 도구: `list_schedules`, `register_schedule`, `register_one_off_schedule`, `delete_schedule`

## 💡 사용 지침

1. **반복 일정 등록 (register_schedule)**: 매분, 매시간, 매일 특정 시간처럼 주기적으로 실행해야 하는 작업(예: "1분마다 알려줘", "5분마다 상태 점검해줘", "매일 아침 9시에 뉴스 브리핑")에 사용합니다.
2. **일회성 알림 등록 (register_one_off_schedule)**: "5분 뒤에 알려줘", "내일 오후 1시에 전화 예약 확인해줘"와 같이 한 번만 실행하고 종료될 작업에 사용합니다.
3. **스케줄 조회 (list_schedules)**: 사용자가 현재 등록된 일정 확인, 수정, 삭제를 원하거나 어떤 스케줄이 있는지 먼저 확인해야 할 때 사용합니다.
4. **스케줄 삭제 (delete_schedule)**: 사용자가 기존 스케줄을 지워 달라고 하면 사용합니다. taskId를 알고 있으면 그대로 삭제하고, 없으면 먼저 `list_schedules`로 현재 활성 에이전트의 등록 스케줄을 확인한 뒤 적절한 스케줄을 특정해서 삭제합니다.
5. **전용 도구 우선**: 스케줄 관련 요청에서는 `bash`로 파일 시스템이나 로그를 직접 뒤지기 전에 반드시 `list_schedules`, `register_schedule`, `register_one_off_schedule`, `delete_schedule` 같은 전용 도구를 우선 사용합니다.
6. **시간 계산 도움**: `register_one_off_schedule`을 사용할 때, 현재 시간으로부터 목표 시간까지의 차이를 '분(minutes)' 단위로 계산하여 입력해야 합니다. 스스로 계산이 어려우면 사용자에게 현재 시각을 물어보거나 추측하지 마세요.
7. **확인 응답**: 일정을 등록하거나 삭제한 후에는 어떤 작업이 어떻게 처리되었는지 사용자에게 명확하게 피드백을 줍니다.
8. **짧은 반복 주기 허용**: "1분마다", "5분마다" 같은 짧은 반복 알림도 지원 대상입니다. 지원되지 않는다고 추측하지 말고 `register_schedule`로 등록을 시도하세요. 예: 1분마다=`* * * * *`, 5분마다=`*/5 * * * *`
