# Scheduling Skill

일정 예약 및 알림 설정을 관리하기 위한 스킬입니다. 반복되는 작업이나 일회성 타이머/알림을 등록, 조회, 삭제할 수 있습니다.

사용 도구: `list_schedules`, `register_schedule`, `delete_schedule`

## 💡 사용 지침

1. **반복 일정 등록 (register_schedule)**: 매분, 매시간, 매일 특정 시간처럼 주기적으로 실행해야 하는 작업(예: "1분마다 알려줘", "5분마다 상태 점검해줘", "매일 아침 9시에 뉴스 브리핑")에 사용합니다.
2. **일회성 알림 (register_schedule, scheduleType: "once")**: "5분 뒤에 알려줘", "내일 오후 1시에 전화 예약 확인해줘"처럼 한 번만 실행할 때는 `scheduleType: "once"`, `startAt: <실행일시 ISO 8601>`으로 등록합니다.
3. **스케줄 조회 (list_schedules)**: 사용자가 현재 등록된 일정 확인, 수정, 삭제를 원하거나 어떤 스케줄이 있는지 먼저 확인해야 할 때 사용합니다.
4. **스케줄 삭제 (delete_schedule)**: 사용자가 기존 스케줄을 지워 달라고 하면 사용합니다. taskId를 알고 있으면 그대로 삭제하고, 없으면 먼저 `list_schedules`로 현재 활성 에이전트의 등록 스케줄을 확인한 뒤 적절한 스케줄을 특정해서 삭제합니다.
5. **날짜/시간 변환**: `startAt`은 ISO 8601 형식(예: `2026-04-13T14:30:00+09:00`)으로 입력합니다. 사용자가 "5분 뒤", "내일 오후 3시" 등 자연어로 말하면 현재 시각(시스템 프롬프트 참조)을 기준으로 계산하여 변환합니다.
6. **시작/종료일 설정**: 반복 스케줄에 `startAt`(시작 일시)과 `endAt`(종료 일시)을 지정할 수 있습니다. "다음 달부터", "이번 주 금요일까지"처럼 기간이 있는 요청은 ISO 8601로 변환해 전달합니다. 생략 시 즉시 시작, 무한 반복입니다.
7. **확인 응답**: 일정을 등록하거나 삭제한 후에는 어떤 작업이 어떻게 처리되었는지 사용자에게 명확하게 피드백을 줍니다.
